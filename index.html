What is Azure.Mobile?

Most mobile apps today require some kind of backend infrastucture and a great deal of those apps use Azure Mobile Services. Many of the apps we've built this year use App Service and we saw an oppertunity to share more code whilst improving the development experiance.

How does it work?

We've create Azure.Mobile as a wrapper around the offical Azure App Service .NET SDKs from Microsoft. This library includes standard boiler-plate code that all developers would need to write in order to interact with an Azure backend.

To make your lives as easy as possible we've created a number of BaseClasses for you to inherit from. All of your models should inherit from our EntityData class in order to ensure they support Offline Sync. You can also inherit from our Forms ViewModel base class. This includes everything you need to bind to your remote tables with only 1 line of code.
